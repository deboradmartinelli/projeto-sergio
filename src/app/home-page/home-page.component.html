PARTIU FAZER UM BACKEND

  • Na tela de personagens, estou chamando uma api pública que devolve para o front-end os personagens do Starwars (http://swapi.dev/api/people)... essa é só um exemplo pra vc ver como funciona uma chamada para uma api
  • Na tela de clientes, é um CRUD completo, pode fazer a festa... cada cliente deve ter essas propriedades (com os seguintes tipos):

String id
String nome
String tipo (pode ser PF ou PJ, legal tentar usar ENUM)
String genero (pode ser M ou F, legal tentar usar ENUM)
String dataNascimento (é uma data, nesse formato ano-mes-dia.... 2024-06-12)
String observacao
Boolean inativo

A chamada está sendo feita para essa url: http://localhost:8080/api/v1/cliente, quando vc estiver fazendo seu backend em Java, e for subir ele pra rodar na sua máquina, provavelmente ele vai estar nessa porta já... mas, se precisar editar, está no arquivo clientes.service.ts deste projeto
